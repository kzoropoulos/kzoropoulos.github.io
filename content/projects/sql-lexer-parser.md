## Overview 
The repository contains the implementation of a compiler front-end, engineering a custom lexical analyzer and parser utilizing the combination of Flex, Bison, and C. It processes **mySQLq**, a domain-specific pseudo-language based on SQL. The project was developed as a laboratory assignment for the "Principles of Programming Languages & Compilers" course at the Computer Engineering and Informatics Department, University of Patras. The core architecture utilizes Flex for token generation (Lexical Analysis) and Bison for syntactic structure validation (Parsing). This is augmented with a custom Symbol Table written in C to enforce strict semantic rules and scope resolution.

## System Architecture
The architecture is strictly modular and comprises three distinct phases:

*   **Lexical Analysis (`askisi.l`)**: Employs deterministic finite automata (DFA) generated by Flex to stream characters into discrete tokens such as keywords, identifiers, literals, and operators. Comments and whitespaces are explicitly discarded during this phase.
*   **Syntax Analysis (`askisi.y`)**: Implements a Context-Free Grammar (CFG) in Backus-Naur Form (BNF) to design strict grammatical rules for the accurate interpretation and robust error handling of SQL instructions. Bison generates a parser that verifies the structural correctness of the mySQLq statements.
*   **Semantic Analysis (`symtable.c`)**: A dynamically allocated Symbol Table data structure maintains the state of declared entities, including tables, columns, types, and aliases. It is invoked during parsing reductions to perform type-checking and scope validation.

## Supported Features & Constraints

### 1. Syntactic Constructs
*   Supports `CREATE TABLE` and `SELECT` statement constructs.
*   Enforces strict sequence clauses (`SELECT` -> `FROM` -> `WHERE` -> `GROUP BY` -> `ORDER BY` -> `LIMIT`).
*   Keywords are case-insensitive, while identifiers are strictly case-sensitive.
*   Requires a mandatory semicolon (`;`) at the end of each statement for termination.

### 2. Semantic Validation
*   **Entity Resolution**: Verifies table and column existence prior to reference and prevents duplicate declarations within the same scope.
*   **Type System**: Enforces strong typing in the `WHERE` clause. `INT` evaluates against integer literals, `FLOAT` evaluates against integer or floating-point literals, and `VARCHAR(n)` evaluates against string literals.
*   **Relational Algebra**: Validates `JOIN ... ON ...` constraints across multiple tables, verifying that the compared tables and columns exist.
*   **Aliasing**: Resolves table aliases (`AS`) and validates aliased column references (e.g., `alias.column_name`).

## System Requirements
To compile and execute the parser, the required toolchain includes:
*   **C Compiler**: `gcc` (GNU Compiler Collection).
*   **Lexer Generator**: `flex`.
*   **Parser Generator**: `bison`.
